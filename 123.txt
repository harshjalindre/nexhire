🚀 Changes Required to Make NexHire a Revenue-Generating SaaS
 
  1. 💰 Billing & Subscription System (Critical — No Revenue Without This)
 
  What's missing: There's zero billing infrastructure. No pricing tiers, no payment processing, no usage metering.
 
  Changes needed:
    • Add a `Subscription` and `Plan` model to Prisma schema (linked to Tenant)
    • Integrate Stripe/Razorpay — create a /api/billing route group for:
       • Plan selection, checkout sessions, webhook handlers for payment events
       • Subscription lifecycle (trial → active → past_due → canceled)
    • Implement plan-based feature gating in the API middleware — e.g.:
       • Free: 1 college, 50 students, 5 drives/month
       • Pro (₹5K–15K/month): 500 students, unlimited drives, reports, CSV import
       • Enterprise (custom): Unlimited everything, SSO, API access, priority support
    • Add a billing dashboard in nexhire-web for college admins (current plan, invoices, upgrade CTA)
    • Usage metering — track student count, drives created, API calls per tenant
 
 
  ──────────────────────────────────────────────────────────────────────────────
 
  2. 🏠 Public Landing Page & Onboarding Funnel
 
  What's missing: No marketing site, no self-service signup. Currently only seed-based demo accounts.
 
  Changes needed:
    • Build a public landing page (marketing site at /) — pricing table, feature highlights, testimonials, CTA
    • Self-service tenant registration — "Sign up your college" flow that:
       • Creates a new Tenant + User (college admin)
       • Starts a 14-day free trial
       • Sends welcome email with onboarding steps
    • Onboarding wizard — guide new college admins through: add company → create drive → invite students
    • Student self-registration with college code (this exists partially, needs polish)
 
 
  ─────────────────────────────────────────────────────────────────────────────────────
 
  3. 🔐 Production-Grade Auth & Security
 
  What's missing: Hardcoded passwords like password123, no email verification, no password reset, no refresh tokens.
 
  Changes needed:
    • Email verification on signup (integrate SendGrid/Resend/AWS SES)
    • Password reset flow (/forgot-password, /reset-password)
    • Refresh token rotation (currently only access tokens — sessions will expire abruptly)
    • Account lockout after failed login attempts
    • SSO/SAML for enterprise tier (Google Workspace, Microsoft Entra)
    • Remove all hardcoded demo credentials from production builds
    • Proper secrets management (env vars via Vault/AWS Secrets Manager, not .env files)
 
 
  ──────────────────────────────────────────────────────────────────────────────────────
 
  4. 📊 Analytics & Reporting (Key Upsell Feature)
 
  What's missing: No analytics dashboards, no placement statistics, no exportable reports.
 
  Changes needed:
    • College Admin dashboard metrics:
       • Placement rate (% students placed), offer distribution by company, avg package, branch-wise stats
       • Drive funnel: applications → shortlisted → selected → offered
    • Super Admin platform metrics:
       • Active tenants, MRR, churn rate, usage per tenant
    • Export capabilities — the Go engine has report stubs (POST /api/reports) but needs:
       • Actual PDF generation (placement letters, summary reports)
       • CSV/Excel exports of student data, drive results
    • Scheduled reports via email (weekly/monthly digest)
 
 
  ───────────────────────────────────────────────────────
 
  5. 📧 Email & Communication System
 
  What's missing: No transactional emails at all. WebSocket notifications exist but no email/SMS.
 
  Changes needed:
    • Transactional email service (SendGrid/Resend):
       • Drive announcements to eligible students
       • Application status updates
       • Welcome emails, password reset
       • Invoice/billing emails
    • Email templates — branded, per-tenant customizable
    • Optional SMS notifications (Twilio) for drive reminders — premium tier feature
 
 
  ──────────────────────────────────────────────────────────────────────────────────
 
  6. 📁 File Storage & Resume Management
 
  What's missing: Resume upload exists in the API routes but likely uses local filesystem.
 
  Changes needed:
    • Cloud file storage (AWS S3/Cloudflare R2) for:
       • Resume PDFs
       • Company logos
       • Bulk CSV uploads
       • Generated reports
    • Signed URL access for secure document retrieval
    • File size limits per plan tier
    • Resume parsing (premium feature) — extract skills, education, experience from uploaded PDFs
 
 
  ───────────────────────────────────────────────────────────────────────────────────────────────
 
  7. 🏗️ Infrastructure for Production
 
  What's missing: Docker Compose is dev-only. No CI/CD, no monitoring, no auto-scaling.
 
  Changes needed:
    • Deploy to a cloud provider — AWS ECS/EKS, GCP Cloud Run, or Railway/Render for quick start
    • CI/CD pipeline (GitHub Actions): lint → test → build → deploy
    • Database: Managed PostgreSQL (RDS/Supabase/Neon) with automated backups
    • Monitoring & alerting: Sentry for errors, Prometheus+Grafana or Datadog for metrics
    • CDN for frontend assets (CloudFront/Cloudflare)
    • SSL/TLS certificates (Let's Encrypt via Caddy or managed load balancer)
    • Auto-scaling for the Go engine under high load
    • Database migrations strategy — Prisma migrate in production (not db push)
 
 
  ─────────────────────────────────────────────────────────────────────────────
 
  8. 🧑‍💼 Recruiter Portal (New Revenue Stream)
 
  What's missing: Companies/recruiters have no direct access — everything is managed by college admins.
 
  Changes needed:
    • Add a `RECRUITER` role to the RBAC system
    • Recruiter dashboard: Post drives to multiple colleges, view applicants, shortlist candidates
    • Recruiter self-registration + verification
    • Monetize: Charge recruiters per-drive or monthly subscription for multi-college access
    • This opens up a two-sided marketplace model (colleges pay for management, recruiters pay for access)
 
 
  ────────────────────────────────────────────────────────────────────────────────────────────────────────
 
  9. 📱 Mobile & Accessibility
 
  What's missing: PWA-ready label exists but no actual service worker, push notifications, or offline support.
 
  Changes needed:
    • Proper PWA with service worker, offline caching, install prompt
    • Push notifications (Web Push API) for drive deadlines, application updates
    • Mobile-responsive testing and optimization
    • Consider React Native app in the future for the student experience
 
 
  ──────────────────────────────────────────────────────────────────────
 
  10. 🛡️ Legal & Compliance
 
  What's missing: No terms of service, privacy policy, data processing agreements, or GDPR considerations.
 
  Changes needed:
    • Terms of Service and Privacy Policy pages
    • Data Processing Agreement (DPA) template for enterprise clients
    • Cookie consent banner
    • Data export/deletion capabilities (GDPR/DPDP Act compliance)
    • Audit logging — the AuditLog model exists, needs to actually capture all admin actions
    • Tenant data isolation audit — verify RLS is bulletproof
